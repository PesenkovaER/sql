## Задание 1. Сезонность продуктов

**Условие:**  
Для каждого типа продукта (`product_type`) определите месяц, в котором он продаётся лучше всего (максимальная сумма продаж). Выведите тип продукта, месяц (в формате ГГГГ-ММ) и максимальную сумму.

**SQL-код:**

```sql
WITH monthly_sales AS (
    SELECT 
        p.product_type,
        DATE_TRUNC('month', s.sales_transaction_date) AS month_date,
        SUM(s.sales_amount) AS total_sales
    FROM sales s
    JOIN products p ON s.product_id = p.product_id
    GROUP BY p.product_type, DATE_TRUNC('month', s.sales_transaction_date)
),
ranked AS (
    SELECT 
        product_type,
        month_date,
        total_sales,
        ROW_NUMBER() OVER (PARTITION BY product_type ORDER BY total_sales DESC) AS rn
    FROM monthly_sales
)
SELECT 
    product_type,
    TO_CHAR(month_date, 'YYYY-MM') AS best_month,
    total_sales AS max_sales_amount
FROM ranked
WHERE rn = 1
ORDER BY product_type;
```

**Пояснение:**  
- Соединены таблицы `sales` и `products` для получения типа продукта.  
- `DATE_TRUNC('month', ...)` усекает дату до первого дня месяца, что позволяет группировать продажи по месяцам.  
- Оконная функция `ROW_NUMBER()` ранжирует месяцы для каждого типа продукта по убыванию суммы продаж.  
- Ранг 1 соответствует месяцу с максимальной суммой.  
- `TO_CHAR` форматирует дату в читаемый вид `ГГГГ-ММ`.

## Задание 2. Покрытие дилеров

**Условие:**  
Найдите дилерский центр, у которого наибольшее количество клиентов в радиусе 100 миль. Выведите `dealership_id` и количество клиентов.

**SQL-код:**

```sql
WITH dealer_customer_distance AS (
    SELECT 
        d.dealership_id,
        c.customer_id,
        point(c.longitude, c.latitude) <@> point(d.longitude, d.latitude) AS distance_miles
    FROM dealerships d
    CROSS JOIN customers c
),
dealer_nearby_counts AS (
    SELECT 
        dealership_id,
        COUNT(customer_id) AS customers_within_100_miles
    FROM dealer_customer_distance
    WHERE distance_miles <= 100
    GROUP BY dealership_id
)
SELECT 
    dealership_id,
    customers_within_100_miles
FROM dealer_nearby_counts
ORDER BY customers_within_100_miles DESC
LIMIT 1;
```

**Пояснение:**  
- Расширения `cube` и `earthdistance` позволяют использовать оператор `<@>` для вычисления расстояния в милях между двумя географическими точками, заданными через конструкцию `point(longitude, latitude)`.  
- `CROSS JOIN` создаёт декартово произведение всех клиентов и дилеров, затем для каждой пары вычисляется расстояние.  
- Фильтр `distance_miles <= 100` оставляет только клиентов в пределах 100 миль от дилерского центра.

## Задание 3. Извлечение из JSON

**Условие:**  
Из таблицы `customer_sales` извлеките поле `sales` (массив внутри JSON), разверните его с помощью `jsonb_array_elements` и посчитайте общую сумму продаж, хранящуюся внутри JSON.

**SQL-код:**

```sql
SELECT SUM((elem ->> 'sales_amount')::numeric) AS total_sales_amount
FROM customer_sales,
     jsonb_array_elements(customer_json -> 'sales') AS elem;
```

**Скриншот результата:**  

*(вставьте сюда скриншот с результатом, например, `267253437.585`)*

**Пояснение:**  
- Оператор `->` извлекает JSON-массив по ключу `sales`.  
- Функция `jsonb_array_elements` разворачивает массив в набор строк, каждая строка содержит один объект из массива.  
- Из каждого объекта извлекается поле `sales_amount` как текст (`->>`), затем приводится к числовому типу и суммируется.  
- Запрос не требует фильтрации строк, так как `jsonb_array_elements(NULL)` не возвращает строк и не вызывает ошибок.  
- Результат – общая сумма всех продаж из всех JSON-массивов во всех записях таблицы `customer_sales`.
